robotmem: Persistent Episodic Memory for Robot Learning

Charles Jia

robotmem.com | founder@robotmem.com

ORCID: 0009-0005-3749-9786

Abstract

Robot learning agents discard accumulated experience after each training episode, preventing knowledge transfer across episodes and environments. We present robotmem, an open-source episodic memory system that provides persistent, queryable experience storage for robot learning. robotmem stores physical experiences as natural language descriptions with structured metadata, enabling hybrid retrieval through full-text search, vector similarity, and spatial sorting via Reciprocal Rank Fusion. In multi-seed experiments on MuJoCo FetchPush-v4, a memory-augmented heuristic policy improves success rate by +8.8 percentage points over baseline (5 seeds, paired t(4) = 3.96, p = 0.017, Cohen's d = 1.77). Cross-environment transfer from FetchPush to FetchSlide yields +9.2pp improvement (5 seeds, t(4) = 4.76, p = 0.009), demonstrating that natural-language-stored experiences can transfer across tasks with different dynamics. An ablation study reveals that recalled experiences benefit the full trajectory, not just the execution phase. robotmem is model-agnostic, requires no external services, and integrates via Model Context Protocol (MCP) or direct Python API.

Keywords: episodic memory, robot learning, experience transfer, persistent memory, retrieval-augmented policy

1. Introduction

A fundamental limitation of standard reinforcement learning is episodic amnesia: agents treat each training episode as isolated, discarding accumulated experience when training ends. Experience replay buffers [1] mitigate this within a single training run, and Hindsight Experience Replay (HER) [9] further enables learning from failed trajectories in multi-goal settings. However, these techniques offer no persistence across training runs and no queryable interface for retrieving past experiences by content or spatial context.

Transfer learning approaches — including progressive neural networks [10], policy distillation [11], and sim-to-real methods — address cross-environment knowledge reuse but operate at the policy level, transferring learned parameters rather than discrete experiences. Retrieval-augmented generation (RAG), widely successful in NLP [12], has not been systematically applied to robot learning as an experience retrieval mechanism.

This limitation is particularly costly in robotics, where physical interaction data is expensive to collect. A robot that has learned to push objects with optimal force and angle must relearn these physical parameters from scratch in every new training session. Human learning, by contrast, accumulates experience across episodes — a cook who burned a dish yesterday adjusts the temperature today.

Recent work has recognized the importance of memory in robot learning. MemoryVLA [2] augments vision-language-action models with memory modules, MemER [3] introduces memory-enhanced robot learning, and EchoVLA [4] echoes past experiences to improve policy performance. However, these approaches are typically model-specific, closed-source, or tightly coupled to particular architectures.

We present robotmem, an open-source episodic memory system designed as a universal infrastructure layer for robot learning. Unlike the above systems, robotmem stores experiences as natural language descriptions — a design choice that enables human inspection, integration with language-model-based agents, and potentially facilitates cross-environment transfer by abstracting over environment-specific details (though our experiments do not yet isolate this factor). Our key contributions are:

  1. A hybrid retrieval method for robot experience recall, combining full-text search (FTS5), vector similarity, and spatial sorting via Reciprocal Rank Fusion (RRF), with structured metadata filtering.
  2. Multi-seed experimental evidence on MuJoCo FetchPush-v4 that memory-augmented heuristic policies outperform the baseline by +8.8 ± 4.4 percentage points (5 seeds), with ablation showing all-phase blending (65.3%) outperforms push-phase-only (59.0%).
  3. Cross-environment transfer from FetchPush to FetchSlide (+9.2 ± 3.9 percentage points, 5 seeds), co-occurring with natural language storage and suggesting — though not yet causally establishing — that language-level abstraction facilitates cross-task transfer.
  4. An open-source system (Apache 2.0) with 7 APIs, integrable via MCP or Python SDK, designed as reusable infrastructure for memory-augmented robot learning research.

2. System Architecture

2.1 Design Principles

robotmem is guided by five design principles:

  1. Machine-actionable: Memories store actionable parameters (forces, angles, strategies), not just observations.
  2. Human-readable: All memories are stored as natural language, making them inspectable and debuggable. We hypothesize that this abstraction level may facilitate cross-environment transfer by abstracting over environment-specific state representations, though our experiments do not yet isolate this factor (Section 6).
  3. Minimal complexity: SQLite-based, no external services, no GPU required.
  4. Fail-loud: Memory recall timeout is soft (degraded but functional); memory write failure is hard (must report error).
  5. Additive extension: New perception modalities are added without modifying existing code.

2.2 Memory Storage

Each memory record contains:

  • content: Natural language description of the experience (e.g., "Approaching from 15° left with 12N force succeeds for centered objects")
  • context: Structured JSON with four sections:
  • params: Physical parameters (approach velocity, grip force, final distance to target)
  • spatial: Spatial coordinates (grip position, object position, target position, scene tag)
  • robot: Robot metadata (type, end effector, degrees of freedom)
  • task: Task outcome (name, success, steps, total reward)
  • embedding: Dense vector representation (384 dimensions, FastEmbed ONNX) for semantic search
  • type: fact (learned experience) or perception (raw sensor data reference)
  • collection: Namespace for experiment isolation

Memories are recorded at episode end: after each episode completes, the system computes episode-level statistics (average approach velocity, final object-target distance, success/failure) and stores both a natural language summary and the structured context. This episode-level granularity trades step-level detail for storage efficiency and retrieval relevance.

Scope clarification: robotmem stores episode-level summaries and parameters as natural language + structured JSON. It does not store raw sensory data (images, point clouds, force-torque trajectories); such data should be stored in domain-specific formats (ROS bags, HDF5) and referenced via the perception type field.

2.3 Hybrid Retrieval

The recall API implements Reciprocal Rank Fusion (RRF) across three retrieval channels. Given a query, each channel produces a ranked list. The fused score for item i is:

RRF(i) = 1/(k + rank_FTS5(i)) + 1/(k + rank_vec(i)) + 1/(k + rank_spatial(i))

where rank_c(i) is the rank of item i in channel c, and k = 60 (standard RRF constant [14]). Channels that do not return item i contribute zero to the sum.

The three channels are:

  1. FTS5 full-text search: SQLite FTS5 with jieba tokenization for CJK support. Matches keyword-level queries efficiently.
  2. Vector similarity: vec0 extension for cosine similarity search on ONNX-embedded vectors (FastEmbed, 384 dimensions).
  3. Spatial sorting: Optional re-ranking by Euclidean distance to a target coordinate, enabling queries like "what worked near position [1.3, 0.7, 0.4]?"

Additional filtering is supported through context_filter, which parses JSON paths (e.g., {"task.success": true}) to retrieve only successful experiences.

2.4 Integration

robotmem provides two integration paths:

  • MCP Server: 7 tools exposed via Model Context Protocol, compatible with any MCP-capable AI agent (Claude, GPT, etc.)
  • Python SDK: Direct import for tighter integration in training loops, bypassing MCP overhead.

3. Experimental Setup

3.1 Environment

We use MuJoCo's FetchPush-v4 [7] via Gymnasium-Robotics [8]: a simulated 7-DOF Fetch robot arm that must push a cube to a target position on a tabletop. The full observation space includes gripper position (3D), gripper linear velocity (3D), object position (3D), object rotational velocity (3D), object-gripper relative position (3D), and object-target relative position (3D). Our heuristic policy uses gripper position, object position, and target position; velocity observations are not used. The action space is 4-dimensional: 3D Cartesian displacement + gripper control. Our policy keeps the gripper open throughout (no grasping), using only the 3D displacement dimensions. An episode succeeds if the object is within 5cm of the target.

3.2 Baseline Policy

We implement a 4-stage heuristic policy as a proof-of-concept baseline:

  1. Rise: Lift gripper above the object (avoid collision during positioning)
  2. Move behind: Navigate to a position behind the object (opposite to push direction), offset 4cm from the object center
  3. Descend: Lower to object height
  4. Push: Apply force toward the target

We use a heuristic rather than a trained RL policy (e.g., SAC+HER) for two reasons: (1) it provides a deterministic baseline unaffected by exploration noise, isolating the effect of memory augmentation; (2) it allows us to validate the memory system independently of any learning algorithm. We note that HER-trained policies achieve >90% success on FetchPush [9]; our ~52% baseline (mean across 5 seeds) is intentionally weaker, as the goal is to measure the memory system's contribution, not to compete with state-of-the-art RL. Integration with learned policies is future work (Section 6).

3.3 Memory-Augmented Policy

The memory-augmented policy extends the baseline by:

  1. At episode start, querying robotmem with the current object and target positions
  2. Filtering for successful experiences (context_filter={"task.success": true})
  3. Sorting results by spatial proximity to the current object position
  4. Extracting average approach velocity from recalled successful experiences
  5. Blending the memory bias with the heuristic action: action = (1-w) × base + w × bias, where w=0.3

We evaluate two blending strategies: (a) push-phase-only (PhaseAwareMemoryPolicy), applying memory bias only during phase 4 (push), and (b) all-phase (MemoryPolicy), applying memory bias during all four phases. Our initial hypothesis was that push-phase-only would be superior, since approach velocities are most relevant during the push. However, ablation results (Section 4.3) show that all-phase blending outperforms push-phase-only, suggesting that recalled experiences also provide beneficial directional information during positioning phases.

The blending weight w=0.3 was selected by manual tuning over values {0.1, 0.2, 0.3, 0.5, 0.7} on a 30-episode validation run; 0.3 yielded the best balance between memory influence and heuristic stability. We acknowledge this is not a rigorous hyperparameter search and note adaptive weighting as future work.

3.4 Three-Phase Protocol

Phase Episodes Strategy Memory Operations Purpose
A 1-100 Heuristic baseline None Baseline performance
B 101-200 Heuristic + memory write learn + save_perception Write control (populate DB)
C 201-300 Heuristic + memory read/write recall + learn Memory utilization

We run 5 independent seeds (fresh database per seed), reporting mean ± standard deviation. Each seed runs all 300 episodes sequentially. This multi-seed protocol ensures that reported improvements are robust to environment randomization rather than artifacts of a single favorable seed.

3.5 Cross-Environment Transfer

To provide preliminary evidence of cross-environment transfer, we run a separate experiment on FetchSlide-v4, where the robot must slide a puck across a table to reach distant targets. FetchSlide differs from FetchPush in several physical parameters: the puck has lower friction coefficient than the cube, target distances are 2-3x greater, and success requires imparting a precise initial impulse (the puck must slide to the target after contact, unlike push where continuous contact is maintained).

The cross-environment experiment uses three conditions, each run for 100 episodes across 5 independent seeds:

  • Baseline: FetchSlide with no memory (100 episodes × 5 seeds)
  • Cross-environment: FetchSlide + FetchPush memories (100 episodes × 5 seeds)
  • Same-environment: FetchSlide + FetchSlide memories (100 episodes × 5 seeds)

For each seed, a fresh set of FetchPush memories is generated (100 episodes) before testing cross-environment transfer. Same-environment memories are generated from a separate FetchSlide writing phase (100 episodes).

4. Results

4.1 FetchPush-v4 (Memory Augmentation)

Phase Strategy Success Rate (5 seeds) Avg Distance
A (ep 1-100) Heuristic baseline 52.4% ± 4.6% 0.084 ± 0.012m
B (ep 101-200) Memory write (heuristic + learn) 47.6% ± 6.2% 0.097 ± 0.019m
C (ep 201-300) Memory read/write (recall + learn) 61.2% ± 4.6% 0.078 ± 0.015m

Key findings:

  • Phase C shows +8.8 ± 4.4 percentage point improvement over Phase A (paired t(4) = 3.96, p = 0.017; Cohen's d = 1.77; Hedges' g = 1.53, large effect). All 5 seeds show Phase C > Phase A (per-seed deltas: +4, +14, +11, +12, +3 pp). The recall mechanism allows the agent to adjust its push velocity based on spatially similar successful experiences. We note that with only 5 seeds (df = 4), statistical power is limited and small-sample effect size estimates tend to be positively biased; larger-scale replication would strengthen this finding.
  • Phase B shows no statistically significant difference from Phase A (paired t(4) = -1.00, p = 0.37). The apparent Phase B improvement in our earlier single-seed pilot was seed-specific variance. The memory write operations (learn + save_perception) do not affect policy behavior, as expected.
  • Average final distance in Phase C (0.078m) is lower than Phase A (0.084m), though the improvement is modest compared to the success rate gain. This suggests memory augmentation primarily helps with task completion (pushing the object close enough to succeed) rather than fine-grained precision.

4.2 Cross-Environment Transfer (FetchPush → FetchSlide)

Condition Success Rate (5 seeds) Delta vs. Baseline
FetchSlide baseline (no memory) 6.6% ± 2.3% —
FetchSlide + FetchPush memories 15.8% ± 2.7% +9.2 ± 3.9pp
FetchSlide + FetchSlide memories 7.0% ± 1.9% +0.4pp (n.s.)

Key findings:

  • Cross-environment transfer (FetchPush → FetchSlide) yields +9.2 ± 3.9 percentage point improvement over baseline (paired t(4) = 4.76, p = 0.009; Cohen's d = 2.13, large effect). Per-seed deltas: +12, +2, +10, +13, +9 pp.
  • Same-environment transfer shows no significant benefit (paired t(4) = 0.21, p = 0.84). Our earlier single-seed pilot showed 16% same-environment success, which multi-seed replication reveals as seed-specific noise. This asymmetry — cross-environment transfer works but same-environment does not — is the most surprising finding and is discussed further in Section 6.
  • The low absolute success rates (6-16%) reflect the inherently harder FetchSlide task and the limitations of our heuristic baseline on sliding tasks. The heuristic was designed for push (continuous contact), not slide (impulse + inertia), limiting the ceiling of memory-augmented performance.

4.3 Ablation: Push-Phase-Only vs. All-Phase Blending

Condition Success Rate (3 seeds)
Baseline (no memory) 49.3% ± 3.3%
Push-phase-only (PhaseAwareMemoryPolicy) 59.0% ± 1.6%
All-phase (MemoryPolicy) 65.3% ± 3.3%

We compare two memory blending strategies: applying the recalled velocity bias only during the push phase (phase 4) versus during all four phases (rise, move-behind, descend, push). The ablation uses 3 seeds (vs. 5 in the main experiments) due to the need to run three conditions per seed; we prioritized additional conditions over additional seeds. The ablation baseline (49.3%) differs from the main experiment Phase A (52.4%) because different seed sets were used; both values are within each other's standard deviation.

Key findings:

  • All-phase blending (65.3%) outperforms push-phase-only (59.0%) by +6.3 percentage points, overturning our initial hypothesis that memory bias should be restricted to the push phase. With only 3 seeds (df = 2), these results constitute descriptive evidence; statistical testing requires additional seeds (t-critical at alpha=0.05, df=2 is 4.303, making significance difficult to achieve).
  • Both memory conditions substantially outperform the baseline (+9.7pp for push-only, +16.0pp for all-phase), with all 3 seeds showing the same ranking (all-phase > push-only > baseline).
  • The all-phase advantage suggests that recalled experiences encode useful information beyond push velocity: approach direction during positioning (phases 1-3) benefits from memories of successful trajectories. This aligns with the cross-environment transfer finding — the approach strategy, not just the push force, is the primary transferable knowledge.

4.4 Qualitative Analysis

Examining recalled memories provides insight into what transfers:

  • Approach direction: Memories consistently encode positioning behind the object relative to the target. The all-phase ablation confirms this directional information benefits the entire trajectory, not just the push.
  • Force calibration: Successful FetchPush memories encode approach velocities (mean 0.15 m/step) that modulate push velocity when blended at w=0.3. In cross-environment transfer, these values provide a directional signal even when the optimal force for FetchSlide differs.

4.5 Discussion: Abstract Memories Transfer Better Than Specific Ones

The most striking finding across our experiments is the asymmetry in cross-environment vs. same-environment transfer on FetchSlide. FetchPush memories — which encode an approach strategy abstracted from a different task — improve FetchSlide performance by +9.2pp, while FetchSlide's own memories provide no benefit. This pattern suggests a counterintuitive principle: memories that are more abstract (from a different but related task) may transfer more effectively than task-specific memories.

We hypothesize three contributing factors:

  1. Abstraction as regularization: FetchPush memories encode general approach behaviors (direction, contact initiation) without task-specific force parameters. When applied to FetchSlide, the irrelevant force details are diluted by the low blending weight (w=0.3), while the transferable directional information remains. FetchSlide memories, by contrast, encode both generalizable direction and task-specific impulse values — but the linear blending mechanism cannot selectively apply one without the other.

  2. Diversity vs. specificity: 100 episodes of FetchPush generate diverse approach strategies across varied object-target configurations. The same number of FetchSlide episodes generates memories optimized for the narrower sliding dynamics, which may overfit to specific configurations encountered during memory accumulation.

  3. Alignment with curriculum learning: This finding echoes curriculum learning literature, where training on simpler tasks before harder ones often outperforms direct training. FetchPush memories effectively provide a "curriculum" of approach behaviors that, when recalled during FetchSlide, bootstrap the agent's positioning without constraining its task-specific execution.

This hypothesis is testable: a controlled experiment comparing (a) NL memories from a different task, (b) numerical vectors from a different task, (c) NL memories from the same task, and (d) numerical vectors from the same task would isolate the contributions of abstraction level and cross-task diversity. We leave this 2×2 factorial design to future work.

5. Related Work

Experience Replay [1]: The foundational technique for reusing transitions within a single training run.

Hindsight Experience Replay (HER) [9]: Enables learning from failed trajectories by relabeling goals post-hoc. HER is the standard method for FetchPush and achieves >90% success rate with SAC. robotmem is complementary: HER optimizes within-run learning efficiency, while robotmem provides cross-run and cross-environment persistence.

Transfer Learning in Robotics: Progressive neural networks [10] enable cross-task transfer by lateral connections. Policy distillation [11] compresses multi-task knowledge. Sim-to-real approaches transfer policies across simulation and physical domains. These methods transfer at the policy parameter level; robotmem transfers at the experience level using natural language, enabling human-readable, architecture-independent transfer.

Retrieval-Augmented Generation (RAG) [12]: Widely successful in NLP for grounding LLM outputs in retrieved documents. robotmem can be viewed as RAG for robotics: retrieving relevant past experiences to augment current decision-making. Unlike NLP RAG where an LLM synthesizes retrieved context, our current implementation uses simple linear blending — a deliberate simplification that trades reasoning capability for latency and determinism.

Semantic Robot Memory: KnowRob [13] provides a knowledge processing framework for robots using ontological representations. robotmem differs in using natural language rather than formal ontologies, trading inference power for flexibility and human readability.

MemoryVLA [2] (Tsinghua, 2025): Augments VLA models with learned memory modules. Model-specific, not transferable across architectures.

MemER [3] (Stanford, 2025): Memory-enhanced robot learning with retrieval-augmented policies. Closest to our approach in using retrieval for policy augmentation, but integrated into a specific learning framework rather than designed as standalone infrastructure.

EchoVLA [4] (Sun Yat-sen, 2025): Echoes past experiences for VLA improvement. Focuses on vision-language models specifically.

DynaMem [5] (NYU, 2025): Dynamic memory for adaptive robot manipulation. Uses point cloud features rather than natural language.

RMBench [6] (HKU, 2025): Benchmarks for robot memory. Complementary to our work — robotmem could serve as a backend for RMBench evaluations.

robotmem differs from these approaches in being (a) model-agnostic, (b) open-source, (c) based on natural language storage for human readability and cross-environment transfer, and (d) requiring no external services.

6. Limitations and Future Work

Limitations:

  • Our evaluation uses a heuristic policy as a proof-of-concept baseline. We have designed a corresponding SAC+HER experiment with RL-Zoo3-verified hyperparameters, but the computational requirements exceeded available resources for this submission. The fully configured experiment script is included in the repository (examples/fetch_push/sac_experiment.py); we estimate ~10 min/seed on GPU hardware. Validation with learned RL policies may yield different effects, as such policies can more flexibly incorporate recalled information.
  • While multi-seed (5 seeds) results show consistent improvement, the effect size (+8.8pp on FetchPush) is modest. Larger effects may emerge with learned policies that can more flexibly incorporate recalled information.
  • The linear blending mechanism (action = (1-w) × base + w × bias) is fundamentally limited: it can only interpolate between the heuristic and memory-derived actions. For tasks requiring qualitatively different action profiles — such as FetchSlide's "strong impulse then release" — linear blending with a fixed w=0.3 cannot produce the required discontinuous behavior. This likely explains the same-environment transfer failure on FetchSlide: recalled Slide memories encode optimal impulse magnitudes, but the blending mechanism cannot faithfully reproduce them.
  • Same-environment transfer on FetchSlide showed no benefit, while cross-environment transfer succeeded. We hypothesize that FetchPush memories transfer a generalizable approach strategy (direction, contact initiation) rather than task-specific force parameters. FetchSlide memories, by contrast, encode impulse-specific parameters that the linear blending mechanism cannot faithfully apply. Disentangling the contributions of approach direction vs. force calibration requires per-channel ablation and is left to future work.
  • The natural language storage format, while enabling transfer and human readability, may lose precision compared to raw numerical storage for fine-grained motor control.
  • The 0.3 memory blending weight is manually tuned and globally fixed; adaptive weighting based on memory relevance and confidence could improve performance.
  • The current system stores episode-level summaries, not step-level trajectories or raw sensory data (images, point clouds, force-torque curves). Full sensory storage is deferred to future work.

Future Work:

  • Integration with learned RL policies (SAC+HER, PPO, TD3) to measure memory augmentation in the standard FetchPush setting. We expect partially-trained policies (50-60% baseline) to benefit most from memory augmentation, as they retain room for improvement while having learned sufficient basic manipulation skills — analogous to our heuristic baseline at ~52%, where memory augmentation yielded +8.8pp. The magnitude of improvement may differ due to the different policy architecture.
  • Retrieval channel ablation: spatial-only vs. vector-only vs. FTS5-only to quantify each channel's contribution
  • Visual and force-torque memory: storing and retrieving image-based and haptic experiences
  • Adaptive memory blending weight based on retrieval confidence
  • Benchmarking on Isaac Gym and PyBullet environments
  • Controlled comparison of natural language vs. raw numerical memory storage to isolate the role of language abstraction in cross-environment transfer
  • Non-linear blending mechanisms (e.g., gated networks, attention-weighted fusion) that can produce discontinuous action profiles required by tasks like FetchSlide
  • Rust SDK for real-time robot control with microsecond-level recall latency

7. Conclusion

We presented robotmem, an open-source episodic memory system that gives robot learning agents persistent, queryable memory across episodes and environments. Multi-seed experiments on MuJoCo FetchPush-v4 (5 seeds) demonstrate that memory augmentation of a heuristic baseline improves success rate by +8.8 ± 4.4 percentage points. Cross-environment transfer from FetchPush to FetchSlide (5 seeds) yields +9.2 ± 3.9pp improvement, providing evidence that natural-language-stored experiences transfer across environments with different dynamics. An ablation study reveals that all-phase memory blending (65.3%) outperforms push-phase-only (59.0%), indicating that recalled experiences benefit the full trajectory, not just the execution phase. These results demonstrate that episodic memory in natural language is a viable knowledge representation for robot learning, complementing rather than competing with learned policies. robotmem is designed as universal infrastructure: model-agnostic, lightweight (SQLite-based), and integrable via MCP or Python SDK. We release the code, experiment scripts, and accompanying documentation to support reproducible research in memory-augmented robot learning.
